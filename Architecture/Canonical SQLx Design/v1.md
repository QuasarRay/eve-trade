# CANONICAL SCHEMA OWNERSHIP

`distributed-backend/src/trade-settlement/migrations/0001_settlement_schema.sql`
is the canonical migration. The Kubernetes migration copy is generated/verified
against it, and `scripts/verify_schema_ownership.py` fails canonical CI on byte
drift. Rust trade-settlement is the sole runtime mutation owner; Market receives
read-only database credentials and cannot mutate settlement state.
