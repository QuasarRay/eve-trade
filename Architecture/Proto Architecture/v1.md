# Proto Architecture

## Status

Current for the Encore refactor.

## Current Contract Boundary

The Go-to-Go Market contract is no longer modeled as protobuf or generated RPC transport. It is
an Encore private API declared in `market/api.go`:

- `market.SubmitTradeGuiInteraction(ctx, *SubmitTradeGuiInteractionRequest)`
- request body: `RawPayload []byte`
- response body: typed JSON-compatible Go fields returned to the UDP gateway

The only retained protobuf contract is the cross-language Rust settlement
boundary:

- source: `proto/eve/trade_settlement/v1/trade_settlement.proto`
- generated Go: `proto/gen/eve/trade_settlement/v1/trade_settlement.pb.go`
- Rust consumer: `distributed-backend/src/trade-settlement`

## Removed Contract Surface

The previous Market protobuf and generated transport packages were removed because
Encore now owns Go-to-Go API declaration, type-safe calls, service discovery, and
transport wiring.

Removed current-state responsibilities:

- generated transport clients and handlers
- manual RPC interceptors and HTTP server registration
- Market service URL configuration
- Go-to-Go protobuf files used only for in-repository transport

## Generation

`buf generate` now emits only the Go protobuf types required by the Rust
trade-settlement gRPC boundary. `buf.gen.yaml` intentionally does not invoke any
Go-to-Go transport generator.

## Rule

Add protobuf only when a real external or cross-language contract exists. Encore
service APIs should use ordinary Go request/response types.
