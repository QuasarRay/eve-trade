# Trade State Lifecycle

## Status

Current for the Encore runtime architecture.

## State Owners

Market reads trade, wallet, item-stack, and idempotency replay snapshots to make
game-rule decisions. It does not mutate authoritative settlement state directly.

Rust `trade-settlement` owns durable mutation of:

- trade rows and trade state
- item stack and item escrow rows
- wallet and wallet escrow rows
- item and wallet ledgers
- settlement batches, attempts, steps, and idempotency records

## Issue

Market validates seller ownership, item stack state, requested quantity, price,
and optional expiry. It then publishes settlement operations to create the trade,
create item escrow, decrement the source item stack, and append item ledger
entries.

## Accept

Market validates trade openness, expiry, escrow availability, buyer/seller
separation, buyer wallet state, seller wallet state, and optional destination
stack compatibility. It publishes settlement operations for item transfer,
wallet escrow/debit/credit, trade remaining quantity or completion, and ledger
entries.

## Cancel

Market validates the issuer and open cancellable trade state. It publishes
settlement operations to cancel the trade, return remaining escrowed items, and
append item ledger entries.

## Idempotency

Market uses request fingerprints to detect completed duplicate requests before
publishing new work. Rust `trade-settlement` enforces durable idempotency during
atomic execution so duplicate Pub/Sub deliveries do not duplicate effects.
